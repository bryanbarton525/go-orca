When generating code examples, always follow this protocol:
1. **Identify the Alternative:** Explicitly state the unsafe/pre-generic alternative (e.g., 'Before Generics: Using `interface{}`').
2. **Demonstrate the Failure:** Show how the alternative fails or requires type assertion/reflection (e.g., `v.(MyType)`).
3. **Introduce the Fix:** Provide the fully compilable, generic solution using type parameters and constraints.
4. **Justify the Superiority:** Conclude the micro-section with a clear, non-negotiable statement like: 'This proves compile-time safety over runtime uncertainty.'
